# 电商购物车结算系统技术文档

## 1. 文档说明

本文档对应当前目录下的 3 个 C++ 实现：

- 正确版（基线）：`checkout_engine_correct.cpp`
- 待测版：`checkout_engine_buggy.cpp`
- 待测注释版：`checkout_engine_test.cpp`（与待测版逻辑一致，按模块补充中文注释，便于走读与评审）

配套示例文件：

- 输入示例：`checkout_input_example.json`
- 成功输出示例：`checkout_output_success_example.json`（与正确版 `checkout_engine_correct.cpp` 对齐）
- 失败输出示例：`checkout_output_fail_example.json`

---

## 2. 功能范围

系统聚焦购物车结算逻辑，不包含注册、登录、商品检索等功能。  
输入为 JSON 结算请求，输出为 JSON 结算结果或错误信息。

**正确版**实现能力概览：

- 商品行金额与小计（单价 × 数量，按规则取整）
- 多商品合并结算，区分普通商品与特价商品（用于部分优惠基数）
- 优惠券：`discount`、`full_reduction`、`fixed_price`、`shipping`
- 运费：普通地区 / 偏远地区、续重、满额包邮（按业务规则）
- 库存与数量等输入校验

**说明**：`checkout_engine_buggy.cpp` 与 `checkout_engine_test.cpp` 为待测实现，行为以代码为准；测试时建议与正确版同输入对比输出。

## 3. 业务规则实现（正确版）

### 3.1 商品与输入校验

- 商品数量必须为正整数
- 价格、重量、库存不能为负数
- 结算前校验 `quantity <= stock`，不足则返回失败

失败响应字段：

- `status`：`FAIL`
- `error_code`：错误码
- `message`：错误说明

### 3.2 金额精度

- 金额统一按两位小数四舍五入
- 关键计算点使用 `round2` 处理

### 3.3 优惠券规则（正确版）

- 支持多张优惠券按顺序处理
- 同类优惠券不可叠加（同 `type` 仅生效一次）
- 已过期优惠券跳过
- 未达到门槛（`min_purchase`）的优惠券跳过
- 折扣券：若不适用特价商品，则仅对普通商品小计金额打折（由 `applicable_to_special` 控制）

### 3.4 运费规则（正确版）

- 普通地区：首重 1kg 内 6 元，续重每 1kg 加 2 元（不足 1kg 按 1kg 计）
- 偏远地区（新疆、西藏）：首重 1kg 内 15 元，续重每 1kg 加 10 元
- 普通地区：优惠后商品金额满 99 元包邮；偏远地区不参与该包邮规则

### 3.5 输出结构（成功）

成功时 `data` 中包含：

- `total_original_price`
- `total_discount`
- `shipping_fee`
- `final_payable`
- `breakdown`（明细）
- `message`

| 文件 | 作用 |
|------|------|
| `checkout_engine_correct.cpp` | 规格与基线实现，供对照预期结果 |

## 5. Python 交互预留

三个文件均可通过 C 导出函数供 Python（`ctypes` / `cffi`）调用：

- 正确版：`checkout_from_json(const char*)`
- 待测版：`checkout_from_json_buggy(const char*)`
- 待测注释版：`checkout_from_json_test(const char*)`

输入、输出均为 UTF-8 JSON 字符串。

## 6. JSON 输入输出示例

### 6.1 输入示例

参考文件：`checkout_input_example.json`

核心结构：

```json
{
  "action": "checkout",
  "data": {
    "user_id": "U8821",
    "region": "Xinjiang",
    "items": [],
    "coupons": []
  }
}
```

### 6.2 成功输出示例

参考文件：`checkout_output_success_example.json`（数值与正确版一致）

```json
{
  "status": "SUCCESS",
  "data": {
    "total_original_price": 318.8,
    "total_discount": 29.9,
    "shipping_fee": 25.0,
    "final_payable": 313.9,
    "breakdown": {
      "items_subtotal": 318.8,
      "coupon_savings": 29.9,
      "shipping_discount": 0.0
    },
    "message": "结算成功"
  }
}
```

### 6.3 失败输出示例

参考文件：`checkout_output_fail_example.json`

```json
{
  "status": "FAIL",
  "error_code": "E_STOCK_001",
  "message": "商品 SKU_001 库存不足，当前库存: 0, 需求: 1"
}
```

## 7. 编译与运行

> 依赖 `nlohmann/json.hpp`（需单独配置头文件路径）。

### 7.1 编译（示例）

```bash
g++ -std=c++17 checkout_engine_correct.cpp -o checkout_correct
g++ -std=c++17 checkout_engine_buggy.cpp -o checkout_buggy
g++ -std=c++17 checkout_engine_test.cpp -o checkout_test
```

### 7.2 运行（示例）

```bash
checkout_correct < checkout_input_example.json
checkout_buggy < checkout_input_example.json
checkout_test < checkout_input_example.json
```

## 8. 测试建议

- 功能测试：正常结算、优惠券门槛、特价商品参与规则、偏远地区运费、包邮条件
- 异常测试：库存不足、非法数量、JSON 格式错误、过期券与未达门槛券
- 对比测试：同一输入下正确版与待测版输出差异分析
- 回归测试：缺陷修复后复跑用例集

## 9. 补充 JSON 样例

### 正常一单（无券）

```json
{
  "action": "checkout",
  "data": {
    "user_id": "U0001",
    "region": "Beijing",
    "items": [
      {"sku_id": "SKU_A", "name": "商品A", "price": 120.0, "quantity": 1, "weight": 1.0, "is_special": false, "stock": 5}
    ],
    "coupons": []
  }
}
```

### 含折扣券（与输入示例同结构）

可直接使用 `checkout_input_example.json` 作为模板修改 `region`、`items`、`coupons`。
